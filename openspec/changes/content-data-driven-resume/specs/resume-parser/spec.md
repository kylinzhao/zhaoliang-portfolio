# 规范：简历解析器

## 新增需求

### 需求:解析 Markdown + YAML Front Matter 文件
系统必须提供解析器函数，能够读取并解析 `content/resume.md` 文件，将其转换为结构化的 TypeScript 对象。

#### 场景:成功解析文件
- **当** 调用解析器函数且 `resume.md` 文件格式正确
- **那么** 系统必须返回包含以下属性的对象：
  - `frontmatter`: YAML Front Matter 解析后的对象（类型：ResumeFrontmatter）
  - `content`: Markdown 正文内容（类型：string）

#### 场景:解析时机
- **当** 应用构建时
- **那么** 系统必须调用解析器函数读取并解析 `resume.md`

---

### 需求:提供类型安全的 API
解析器必须使用 TypeScript 类型定义，确保编译时和运行时的类型安全。

#### 场景:函数签名
- **当** 调用解析器
- **那么** 函数签名必须是：`getResumeData(): Promise<ParsedResume>`
  - 返回类型：`Promise<ParsedResume>`
  - 包含类型定义的 `frontmatter` 和 `content`

#### 场景:类型导出
- **当** 其他模块导入解析器
- **那么** 系统必须导出 `ResumeFrontmatter` 和 `ParsedResume` 类型定义

---

### 需求:支持同步和异步 API
系统必须同时提供同步和异步版本的 API，以适应不同的使用场景。

#### 场景:同步 API
- **当** 需要同步读取数据（例如在 Node.js 环境）
- **那么** 必须提供 `getResumeDataSync()` 同步函数

#### 场景:异步 API
- **当** 需要异步读取数据（例如在 Next.js Server Component）
- **那么** 必须提供 `getResumeData()` 异步函数

---

### 需求:错误处理机制
解析器必须包含完善的错误处理机制，捕获并报告各种解析错误。

#### 场景:文件读取错误
- **当** 文件系统读取失败（文件不存在、权限不足等）
- **那么** 必须抛出包含错误信息的 Error 对象

#### 场景:YAML 解析错误
- **当** YAML Front Matter 格式错误
- **那么** 必须抛出包含具体错误位置的 Error 对象

#### 场景:编码错误
- **当** 文件编码不是 UTF-8
- **那么** 必须抛出编码错误的 Error 对象

---

### 需求:数据验证
解析器必须使用 Zod schema 验证解析后的数据，确保数据完整性。

#### 场景:数据验证
- **当** 解析完成后
- **那么** 系统必须使用 Zod schema 验证 `frontmatter` 数据

#### 场景:验证失败
- **当** 验证失败
- **那么** 必须抛出 `ZodError`，包含具体的验证错误信息

#### 场景:验证成功
- **当** 验证成功
- **那么** 必须返回经过类型断言的数据对象

---

### 需求:缓存机制
解析器必须实现缓存机制，避免重复解析同一文件。

#### 场景:开发环境缓存
- **当** 在开发环境下多次调用解析器
- **那么** 系统可以使用模块级缓存（首次解析后缓存结果）

#### 场景:生产环境缓存
- **当** 在生产环境下
- **那么** 解析仅在构建时执行一次，无需运行时缓存

---

### 需求:提供示例文件
系统必须提供 `resume.example.md` 示例文件，展示正确的数据格式。

#### 场景:示例文件位置
- **当** 查看项目根目录
- **那么** 必须存在 `content/resume.example.md` 文件

#### 场景:示例文件内容
- **当** 查看 `resume.example.md`
- **那么** 必须包含所有必需字段和可选字段的完整示例

---

### 需求:热更新支持
在开发环境下，系统必须支持热更新功能。

#### 场景:文件变化检测
- **当** 开发环境下 `resume.md` 文件发生变化
- **那么** 系统必须自动重新解析文件并刷新页面

#### 场景:热更新不影响生产环境
- **当** 在生产环境下
- **那么** 必须不启用热更新功能
