# 规范：动效系统

## 新增需求

### 需求:Hero 区域 Stagger 进场动画
网站首次加载时，Hero 区域的文本元素必须以交错方式依次淡入上升。

#### 场景:首次加载时的 Stagger 动画
- **当** 用户首次访问网站
- **那么** Hero 区域的文本元素必须依次执行以下动画：
  - 姓名淡入上升（延迟 0s）
  - 职位淡入上升（延迟 0.1s）
  - 核心技能淡入上升（延迟 0.2s）
  - 按钮淡入上升（延迟 0.3s）
- **并且** 每个元素的动画时长必须为 0.6s
- **并且** 必须使用 ease-out 缓动函数
- **并且** 初始透明度必须为 0
- **并且** 最终透明度必须为 1
- **并且** 初始 Y 轴位置必须为 20px
- **并且** 最终 Y 轴位置必须为 0

---

### 需求:滚动触发淡入动画
当页面区块进入视口时，必须触发淡入动画。

#### 场景:滚动到 About 区域
- **当** 用户滚动到 About 区域且该区域至少 20% 可见
- **那么** About 区域的内容必须从透明度 0 淡入到透明度 1
- **并且** 动画时长必须为 0.6s
- **并且** 必须使用 ease-out 缓动函数

#### 场景:滚动到核心成就区域
- **当** 用户滚动到核心成就区域且该区域至少 20% 可见
- **那么** 核心成就卡片必须依次从左到右淡入
- **并且** 每个卡片的延迟增加 0.1s
- **并且** 动画时长必须为 0.6s

#### 场景:滚动到工作经历区域
- **当** 用户滚动到工作经历区域且该区域至少 20% 可见
- **那么** 工作经历项必须从上到下依次淡入
- **并且** 每个经历项的延迟增加 0.15s
- **并且** 动画时长必须为 0.6s

#### 场景:滚动到技术栈区域
- **当** 用户滚动到技术栈区域且该区域至少 20% 可见
- **那么** 技术卡片必须依次从左到右淡入
- **并且** 每个卡片的延迟增加 0.1s
- **并且** 动画时长必须为 0.6s

---

### 需求:卡片悬停动效
当用户将鼠标悬停在卡片上时，必须有微妙的视觉反馈。

#### 场景:悬停在核心成就卡片上
- **当** 用户将鼠标悬停在核心成就卡片上
- **那么** 卡片必须轻微放大到 1.02 倍
- **并且** 边框颜色亮度必须提升
- **并且** 过渡动画时长必须为 0.2s
- **并且** 必须使用 ease-out 缓动函数

#### 场景:鼠标移开核心成就卡片
- **当** 用户将鼠标从核心成就卡片上移开
- **那么** 卡片必须恢复到原始大小
- **并且** 边框颜色必须恢复到原始颜色
- **并且** 过渡动画时长必须为 0.2s

#### 场景:悬停在技术栈卡片上
- **当** 用户将鼠标悬停在技术栈卡片上
- **那么** 卡片必须轻微放大到 1.02 倍
- **并且** 边框颜色亮度必须提升
- **并且** 过渡动画时长必须为 0.2s

#### 场景:鼠标移开技术栈卡片
- **当** 用户将鼠标将鼠标从技术栈卡片上移开
- **那么** 卡片必须恢复到原始大小
- **并且** 边框颜色必须恢复到原始颜色
- **并且** 过渡动画时长必须为 0.2s

---

### 需求:按钮悬停动效
当用户将鼠标悬停在按钮上时，必须有视觉反馈。

#### 场景:悬停在主按钮上
- **当** 用户将鼠标悬停在"下载简历 PDF"或"发送邮件"按钮上
- **那么** 按钮必须轻微放大到 1.02 倍
- **并且** 按钮必须沿 Y 轴向上移动 2px
- **并且** 过渡动画时长必须为 0.2s

#### 场景:鼠标移开主按钮
- **当** 用户将鼠标从按钮上移开
- **那么** 按钮必须恢复到原始大小和位置
- **并且** 过渡动画时长必须为 0.2s

---

### 需求:导航链接悬停动效
当用户将鼠标悬停在导航链接上时，必须有视觉反馈。

#### 场景:悬停在导航链接上
- **当** 用户将鼠标悬停在导航链接上
- **那么** 链接颜色必须变亮
- **并且** 过渡动画时长必须为 0.2s

#### 场景:鼠标移开导航链接
- **当** 用户将鼠标从导航链接上移开
- **那么** 链接颜色必须恢复到原始颜色
- **并且** 过渡动画时长必须为 0.2s

---

### 需求:尊重用户动画偏好
网站必须检测并尊重用户的减少动画偏好设置。

#### 场景:用户启用减少动画偏好
- **当** 用户在操作系统中启用了"减少动画"偏好设置
- **那么** 网站必须禁用所有动画效果
- **并且** 内容必须立即显示，无过渡动画

#### 场景:用户未启用减少动画偏好
- **当** 用户未启用"减少动画"偏好设置
- **那么** 网站必须正常显示所有动画效果

---

### 需求:平滑滚动
点击导航链接时，页面必须平滑滚动到目标区域。

#### 场景:点击导航链接
- **当** 用户点击导航栏中的任意链接
- **那么** 页面必须平滑滚动到对应区域
- **并且** 滚动时长必须根据滚动距离自动计算
- **并且** 必须使用 ease-in-out 缓动函数

---

### 需求:主题切换动画
切换主题时，所有颜色变化必须有平滑过渡。

#### 场景:切换到 Dark 主题
- **当** 用户从 Light 主题切换到 Dark 主题
- **那么** 背景色、文本色、边框色必须同时过渡
- **并且** 过渡时长必须为 0.5s
- **并且** 必须使用 ease-in-out 缓动函数

#### 场景:切换到 Light 主题
- **当** 用户从 Dark 主题切换到 Light 主题
- **那么** 背景色、文本色、边框色必须同时过渡
- **并且** 过渡时长必须为 0.5s
- **并且** 必须使用 ease-in-out 缓动函数

---

### 需求:动画性能优化
所有动画必须使用 GPU 加速以避免性能问题。

#### 场景:验证动画性能
- **当** 任何动画执行时
- **那么** 必须使用 transform 和 opacity 属性（GPU 加速）
- **并且** 禁止使用 width、height、left、top 等触发布局重排的属性
