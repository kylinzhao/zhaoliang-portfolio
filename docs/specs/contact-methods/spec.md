# 规范：联系方式展示

## 新增需求

### 需求:展示邮箱地址
网站必须在联系方式区域展示邮箱地址。

#### 场景:查看邮箱地址
- **当** 用户滚动到联系方式区域
- **那么** 系统必须显示邮箱图标和邮箱地址："zhaoliang.js@gmail.com"
- **并且** 邮箱图标必须使用 lucide-react 的 Mail 图标
- **并且** 邮箱地址必须是可点击的链接

#### 场景:点击邮箱地址
- **当** 用户点击邮箱地址
- **那么** 系统必须打开用户默认邮件客户端
- **并且** 收件人地址必须自动填充为 zhaoliang.js@gmail.com
- **并且** 邮件链接必须使用 `mailto:` 协议

---

### 需求:展示电话号码
网站必须在联系方式区域展示电话号码。

#### 场景:查看电话号码
- **当** 用户滚动到联系方式区域
- **那么** 系统必须显示电话图标和电话号码："18610384127"
- **并且** 电话图标必须使用 lucide-react 的 Phone 图标
- **并且** 电话号码必须是可点击的链接

#### 场景:点击电话号码（移动端）
- **当** 用户在移动设备上点击电话号码
- **那么** 系统必须发起拨号呼叫 18610384127
- **并且** 电话链接必须使用 `tel:` 协议

#### 场景:点击电话号码（桌面端）
- **当** 用户在桌面设备上点击电话号码
- **那么** 系统必须复制电话号码到剪贴板
- **或者** 系统可以显示提示信息告知用户手动拨打电话

---

### 需求:下载简历 PDF
Hero 区域必须提供简历 PDF 下载按钮。

#### 场景:查看下载简历按钮
- **当** 用户查看 Hero 区域
- **那么** 系统必须显示"下载简历 PDF"按钮
- **并且** 按钮样式必须为主按钮样式（背景色 + 边框）

#### 场景:点击下载简历按钮
- **当** 用户点击"下载简历 PDF"按钮
- **那么** 系统必须开始下载简历 PDF 文件
- **并且** PDF 文件路径必须为 `/赵梁-大前端简历.pdf`
- **并且** 文件必须从 public 目录提供

---

### 需求:发送邮件快捷入口
Hero 区域必须提供发送邮件快捷按钮。

#### 场景:查看发送邮件按钮
- **当** 用户查看 Hero 区域
- **那么** 系统必须显示"发送邮件"按钮
- **并且** 按钮样式必须为次按钮样式（透明背景 + 边框）

#### 场景:点击发送邮件按钮
- **当** 用户点击"发送邮件"按钮
- **那么** 系统必须打开用户默认邮件客户端
- **并且** 收件人地址必须自动填充为 zhaoliang.js@gmail.com
- **并且** 邮件主题可以预填充为"简历咨询"（可选）

---

### 需求:联系方式区域布局
联系方式区域必须在页面底部以清晰的布局展示。

#### 场景:联系方式区域布局
- **当** 用户查看联系方式区域
- **那么** 邮箱和电话必须水平排列（桌面端）
- **或者** 邮箱和电话必须垂直排列（移动端）
- **并且** 每个联系方式必须包含图标和文本
- **并且** 图标和文本之间必须有适当的间距

#### 场景:联系方式响应式布局
- **当** 屏幕宽度 ≥ 768px
- **那么** 邮箱和电话必须水平排列
- **当** 屏幕宽度 < 768px
- **那么** 邮箱和电话必须垂直排列

---

### 需求:联系方式的悬停效果
当用户悬停在联系方式上时，必须有视觉反馈。

#### 场景:悬停在邮箱上
- **当** 用户将鼠标悬停在邮箱地址上
- **那么** 邮箱地址必须变亮
- **并且** 鼠标指针必须变为 pointer
- **并且** 过渡动画时长必须为 0.2s

#### 场景:悬停在电话号码上
- **当** 用户将鼠标悬停在电话号码上
- **那么** 电话号码必须变亮
- **并且** 鼠标指针必须变为 pointer
- **并且** 过渡动画时长必须为 0.2s

---

### 需求:联系方式的可访问性
联系方式必须符合可访问性标准。

#### 场景:邮箱链接的可访问性
- **当** 邮箱链接获得键盘焦点
- **那么** 必须显示明显的焦点指示器（outline）
- **并且** 邮箱链接必须可以通过 Enter 或 Space 键激活

#### 场景:电话链接的可访问性
- **当** 电话链接获得键盘焦点
- **那么** 必须显示明显的焦点指示器（outline）
- **并且** 电话链接必须可以通过 Enter 或 Space 键激活

---

### 需求:简历 PDF 文件可用性
简历 PDF 文件必须正确放置在 public 目录中。

#### 场景:验证 PDF 文件存在
- **当** 用户点击下载简历按钮
- **那么** 系统必须成功从 `/赵梁-大前端简历.pdf` 获取文件
- **并且** 如果文件不存在，系统必须显示 404 错误

#### 场景:PDF 文件命名
- **当** 用户下载简历 PDF
- **那么** 下载的文件名必须为"赵梁-大前端简历.pdf"
- **或者** 下载的文件名必须为"zhao-liang-resume.pdf"（可选）

---

### 需求:联系方式的视觉一致性
联系方式的样式必须与整体设计保持一致。

#### 场景:Dark 主题下的联系方式
- **当** 系统处于 Dark 主题
- **那么** 联系方式文本颜色必须为 #fafafa（主文本色）
- **并且** 图标颜色必须为 #a1a1aa（次要文本色）

#### 场景:Light 主题下的联系方式
- **当** 系统处于 Light 主题
- **那么** 联系方式文本颜色必须为 #09090b（主文本色）
- **并且** 图标颜色必须为 #71717a（次要文本色）
