# 规范：核心内容展示模块

## 新增需求

### 需求:展示 Hero 区域
网站首页必须在顶部展示 Hero 区域，包含姓名、职位和核心技能信息。

#### 场景:访问首页查看 Hero 区域
- **当** 用户访问网站首页
- **那么** 系统必须在页面顶部显示 Hero 区域
- **并且** Hero 区域必须包含以下内容：
  - 姓名："赵梁"
  - 职位："大前端技术负责人 · 27人团队管理"
  - 核心技能列表："React · React Native · Next.js" 和 "跨端架构 · SSR/SEO · 工程化"
  - "下载简历 PDF" 按钮
  - "发送邮件" 按钮

#### 场景:点击下载简历按钮
- **当** 用户点击"下载简历 PDF"按钮
- **那么** 系统必须开始下载简历 PDF 文件（/public/赵梁-大前端简历.pdf）

#### 场景:点击发送邮件按钮
- **当** 用户点击"发送邮件"按钮
- **那么** 系统必须打开用户默认邮件客户端
- **并且** 收件人自动填充为 zhaoliang.js@gmail.com

### 需求:展示核心成就
网站必须使用 Bento Grid 布局展示4个核心成就指标。

#### 场景:查看核心成就
- **当** 用户滚动到核心成就区域
- **那么** 系统必须以 Bento Grid 布局展示4个指标卡片：
  1. "📉 62%" - "成本下降"
  2. "📈 72万" - "SEO收录"
  3. "👥 27人" - "团队规模"
  4. "🤖 184" - "AI CR"

#### 场景:核心成就响应式布局
- **当** 屏幕宽度 ≥ 1024px（桌面端）
- **那么** 核心成就卡片必须排列为4列网格
- **当** 屏幕宽度 ≥ 768px 且 < 1024px（平板端）
- **那么** 核心成就卡片必须排列为2列网格
- **当** 屏幕宽度 < 768px（移动端）
- **那么** 核心成就卡片必须排列为单列

### 需求:展示技术栈
网站必须使用 Bento Grid 布局展示核心技术栈。

#### 场景:查看技术栈
- **当** 用户滚动到技术栈区域
- **那么** 系统必须以 Bento Grid 布局展示技术卡片（共13个）：
  1. "React" - "跨端架构核心" (官方 logo)
  2. "React Native" - "多端同构方案" (官方 logo)
  3. "Next.js" - "SSR/SEO专家" (官方 logo)
  4. "Node.js" - "全栈开发" (官方 logo)
  5. "TypeScript" - "类型安全" (官方 logo)
  6. "Tailwind CSS" - "工程化" (官方 logo)
  7. "数据分析" - "数据驱动决策" (通用图标)
  8. "Tableau" - "可视化分析" (官方 logo)
  9. "6Sigma" - "流程优化" (通用图标)
  10. "SEO" - "搜索引擎优化" (通用图标)
  11. "SSR" - "服务端渲染" (通用图标)
  12. "Dify" - "AI应用平台" (通用图标)
  13. "Agent" - "AI智能体" (通用图标)

#### 场景:技术栈图标来源
- **当** 技术有官方品牌 logo
- **那么** 系统必须使用 Simple Icons CDN (cdn.simpleicons.org) 提供的官方 logo
- **并且** logo 必须使用品牌官方颜色
- **当** 技术无官方 logo（概念性技能）
- **那么** 系统必须使用精心设计的 SVG 图标
- **并且** 图标必须继承主题颜色 (currentColor)

#### 场景:技术栈响应式布局
- **当** 屏幕宽度 ≥ 1024px（桌面端）
- **那么** 技术卡片必须排列为3列网格
- **当** 屏幕宽度 ≥ 768px 且 < 1024px（平板端）
- **那么** 技术卡片必须排列为3列网格
- **当** 屏幕宽度 ≥ 640px 且 < 768px（大屏手机）
- **那么** 技术卡片必须排列为2列网格
- **当** 屏幕宽度 < 640px（移动端）
- **那么** 技术卡片必须排列为2列网格

网站必须在工作经历后展示负责过的 App 列表。

#### 场景:查看 App 列表
- **当** 用户滚动到 App 展示区域
- **那么** 系统必须以网格布局展示 App 图标卡片
- **并且** 必须包含以下 App：
  1. "瓜子二手车" - 链接到 App Store
  2. "去哪儿旅行" - 链接到 App Store
  3. "便利蜂" - 链接到 App Store
  4. "车优多" - 链接到 App Store
  5. "车速拍" - 待添加
  6. "车好卖" - 待添加
  7. "检瓜子" - 待添加
  8. "去哪儿睡" - 待添加
  9. "揣着网" - 待添加

#### 场景:App 图标展示
- **当** App 有 icon URL
- **那么** 系统必须显示真实的 App icon
- **当** App icon 为空或加载失败
- **那么** 系统必须显示降级图标（App 名称前两个字）
- **并且** 使用渐变背景和主题色文字

#### 场景:点击 App 卡片
- **当** 用户点击 App 卡片
- **那么** 系统必须在新标签页打开 App Store 链接
- **并且** 添加 `rel="noopener noreferrer"` 安全属性

#### 场景:App 卡片悬停效果
- **当** 用户将鼠标悬停在 App 卡片上
- **那么** 必须显示半透明黑色遮罩层
- **并且** 显示外部链接图标 (ExternalLink)
- **并且** 过渡动画时长为 0.3s

#### 场景:App 展示响应式布局
- **当** 屏幕宽度 ≥ 1024px（桌面端）
- **那么** App 图标必须排列为9列网格
- **当** 屏幕宽度 ≥ 768px 且 < 1024px（平板端）
- **那么** App 图标必须排列为5列网格
- **当** 屏幕宽度 ≥ 640px 且 < 768px（大屏手机）
- **那么** App 图标必须排列为4列网格
- **当** 屏幕宽度 < 640px（移动端）
- **那么** App 图标必须排列为3列网格

### 需求:Hero 区域 Stagger 进场动画
网站加载时，Hero 区域内容必须以交错动画方式依次淡入上升。

#### 场景:页面加载时的动画
- **当** 用户首次访问网站
- **那么** Hero 区域的文本元素必须依次执行淡入上升动画
- **并且** 每个元素的动画延迟增加 0.1s
- **并且** 动画时长为 0.6s
- **并且** 使用 ease-out 缓动函数

---
