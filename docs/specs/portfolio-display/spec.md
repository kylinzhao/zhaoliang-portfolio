# 规范：核心内容展示模块

## 新增需求

### 需求:展示 Hero 区域
网站首页必须在顶部展示 Hero 区域，包含姓名、职位和核心技能信息。

#### 场景:访问首页查看 Hero 区域
- **当** 用户访问网站首页
- **那么** 系统必须在页面顶部显示 Hero 区域
- **并且** Hero 区域必须包含以下内容：
  - 姓名："赵梁"
  - 职位："大前端技术负责人 · 27人团队管理"
  - 核心技能列表："React · React Native · Next.js" 和 "跨端架构 · SSR/SEO · 工程化"
  - "下载简历 PDF" 按钮
  - "发送邮件" 按钮

#### 场景:点击下载简历按钮
- **当** 用户点击"下载简历 PDF"按钮
- **那么** 系统必须开始下载简历 PDF 文件（/public/赵梁-大前端简历.pdf）

#### 场景:点击发送邮件按钮
- **当** 用户点击"发送邮件"按钮
- **那么** 系统必须打开用户默认邮件客户端
- **并且** 收件人自动填充为 zhaoliang.js@gmail.com

### 需求:展示核心成就
网站必须使用 Bento Grid 布局展示4个核心成就指标。

#### 场景:查看核心成就
- **当** 用户滚动到核心成就区域
- **那么** 系统必须以 Bento Grid 布局展示4个指标卡片：
  1. "📉 62%" - "成本下降"
  2. "📈 72万" - "SEO收录"
  3. "👥 27人" - "团队规模"
  4. "🤖 184" - "AI CR"

#### 场景:核心成就响应式布局
- **当** 屏幕宽度 ≥ 1024px（桌面端）
- **那么** 核心成就卡片必须排列为4列网格
- **当** 屏幕宽度 ≥ 768px 且 < 1024px（平板端）
- **那么** 核心成就卡片必须排列为2列网格
- **当** 屏幕宽度 < 768px（移动端）
- **那么** 核心成就卡片必须排列为单列

### 需求:Hero 区域 Stagger 进场动画
网站加载时，Hero 区域内容必须以交错动画方式依次淡入上升。

#### 场景:页面加载时的动画
- **当** 用户首次访问网站
- **那么** Hero 区域的文本元素必须依次执行淡入上升动画
- **并且** 每个元素的动画延迟增加 0.1s
- **并且** 动画时长为 0.6s
- **并且** 使用 ease-out 缓动函数

---

### 需求:核心成就卡片悬停效果
核心成就卡片在鼠标悬停时必须有视觉反馈。

#### 场景:悬停在成就卡片上
- **当** 用户将鼠标悬停在核心成就卡片上
- **那么** 卡片必须轻微放大（scale: 1.02）
- **并且** 边框亮度提升
- **并且** 过渡动画时长为 0.2s
