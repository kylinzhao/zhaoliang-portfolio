# 规范：响应式布局系统

## 新增需求

### 需求:移动端布局适配
网站必须在移动设备（屏幕宽度 < 768px）上正确显示。

#### 场景:移动端导航栏
- **当** 屏幕宽度 < 768px
- **那么** 导航栏链接必须隐藏
- **并且** 仅显示网站名称和主题切换按钮

#### 场景:移动端 Hero 区域
- **当** 屏幕宽度 < 768px
- **那么** Hero 区域必须使用单列布局
- **并且** 姓名字号必须为 text-4xl（36px）
- **并且** 职位字号必须为 text-lg（18px）
- **并且** 按钮必须垂直堆叠

#### 场景:移动端核心成就
- **当** 屏幕宽度 < 640px
- **那么** 核心成就卡片必须排列为2列网格
- **并且** 每个卡片占据 50% 宽度
- **并且** 卡片间距必须为 gap-4 (1rem)
- **并且** 卡片内边距必须为 p-6 (1.5rem)

#### 场景:移动端技术栈
- **当** 屏幕宽度 < 640px
- **那么** 技术卡片必须排列为2列网格
- **并且** 每个卡片占据 50% 宽度
- **并且** 卡片间距必须为 gap-4 (1rem)
- **并且** 卡片内边距必须为 p-4 (1rem)

#### 场景:移动端 App 展示
- **当** 屏幕宽度 < 640px
- **那么** App 图标必须排列为3列网格
- **并且** 每个图标占据 33.333% 宽度

#### 场景:移动端公司 Logo
- **当** 屏幕宽度 < 640px
- **那么** 公司 logo 必须排列为3列网格
- **并且** 每个 logo 占据 33.333% 宽度
- **并且** logo 容器尺寸必须为 56px × 56px

#### 场景:移动端工作经历
- **当** 屏幕宽度 < 768px
- **那么** 工作经历必须使用单列布局
- **并且** 时间戳和职位必须垂直排列
- **并且** 内容左右内边距必须为 1rem

---

- **那么** 核心成就卡片必须排列为2列网格

#### 场景:平板端技术栈

- **那么** 技术卡片必须排列为3列网格
- **并且** 每个卡片占据 33.333% 宽度

#### 场景:平板端 App 展示

- **那么** App 图标必须排列为5列网格

#### 场景:平板端公司 Logo

- **那么** 公司 logo 必须排列为4列网格

### 需求:桌面端布局适配
网站必须在桌面设备（屏幕宽度 ≥ 1024px）上正确显示。

#### 场景:桌面端导航栏
- **当** 屏幕宽度 ≥ 1024px
- **那么** 导航栏必须显示所有链接
- **并且** 导航栏内容必须居中显示
- **并且** 最大内容宽度必须为 1024px

#### 场景:桌面端 Hero 区域
- **当** 屏幕宽度 ≥ 1024px
- **那么** Hero 区域必须使用居中布局
- **并且** 姓名字号必须为 text-7xl（72px）
- **并且** 职位字号必须为 text-2xl（24px）

#### 场景:桌面端核心成就
- **当** 屏幕宽度 ≥ 1024px
- **那么** 核心成就卡片必须排列为4列网格
- **并且** 每个卡片占据 25% 宽度

#### 场景:桌面端技术栈
- **当** 屏幕宽度 ≥ 1024px
- **那么** 技术卡片必须排列为3列网格
- **并且** 每个卡片占据 33.333% 宽度

#### 场景:桌面端 App 展示
- **当** 屏幕宽度 ≥ 1024px
- **那么** App 图标必须排列为9列网格
- **并且** 每个图标占据 11.111% 宽度

#### 场景:桌面端公司 Logo
- **当** 屏幕宽度 ≥ 768px（中屏及以上）
- **那么** 公司 logo 必须排列为5列网格
- **并且** 每个 logo 占据 20% 宽度

---

### 需求:容器最大宽度
桌面端内容区域必须有最大宽度限制以保证可读性。

#### 场景:桌面端内容容器
- **当** 屏幕宽度 ≥ 1024px
- **那么** 内容区域最大宽度必须为 1024px
- **并且** 内容必须水平居中
- **并且** 左右内边距必须为 2rem

---

### 需求:触摸目标大小
移动端的所有可点击元素必须有足够大的触摸区域。

#### 场景:按钮触摸区域
- **当** 屏幕宽度 < 768px
- **那么** 所有按钮的最小触摸区域必须为 44px × 44px
- **并且** 导航链接的最小触摸区域必须为 44px × 44px

---

### 需求:文本可读性
所有屏幕尺寸下的文本必须保持良好的可读性。

#### 场景:行高设置
- **当** 在任何屏幕尺寸下
- **那么** 正文文本的行高必须为 1.75
- **并且** 标题文本的行高必须为 1.2

#### 场景:文本对比度
- **当** 在任何屏幕尺寸下
- **那么** 文本与背景的对比度必须符合 WCAG AA 标准（≥ 4.5:1）

---

### 需求:图片响应式
所有图片必须在各种屏幕尺寸下正确显示。

#### 场景:图片自动缩放
- **当** 图片容器宽度变化
- **那么** 图片必须自动缩放以适应容器
- **并且** 保持原始宽高比

---

### 需求:横向滚动禁用
网站必须禁止意外的横向滚动。

#### 场景:验证无横向滚动
- **当** 用户在任何屏幕尺寸下浏览网站
- **那么** 禁止出现横向滚动条
- **并且** 所有内容必须在视口宽度内正确显示

---

### 需求:视口元标签
网站必须包含正确的视口元标签以支持移动设备。

#### 场景:视口配置
- **当** 网站在任何移动设备上加载
- **那么** 必须包含视口元标签：`<meta name="viewport" content="width=device-width, initial-scale=1">`
